Introduo

O que so grafos?

Um grafo um tipo de estrutura de dados no-tabular, composta por entidades (ns ou vrtices), em que os relacionamentos entre as entidades so demonstrados por arestas.

T... mas qual a diferena entre grafos e rvores?

Grafos possuem uma estrutura mais relaxada do que rvores:

rvores possuem uma aridade, que o nmero mximo de ns filhos que cada n pode possuir. Esta restrio no se aplica a grafos

Grafos podem conter ciclos

Grafos podem ser direcionados ou no-direcionados, enquanto rvores so sempre direcionadas

Direo em grafos

Existem dois tipos de grafos: direcionados e no-direcionados

Em um grafo direcionado, o sentido de movimento pode ser apenas na direo das setas. Esta restrio no existe em um grafo no-direcionado

Quais as caractersticas das arestas?

Geralmente, as arestas de um grafo possuem pesos. O peso um valor que determina o custo de deslocamento de um n outro

Pensando em um exemplo de deslocamento entre cidades, o peso de uma aresta entre as cidades Santa Maria e Cachoeira do Sul seria a distncia em kms destas cidades

Implementao como lista encadeada

Uma implementao em lista em C de grafos a seguinte:

#include <stdio.h>
#include <stdlib.h>

struct Node {
    char *nome;
    int n_vizinhos;
    int *custos;
    struct Node* vizinhos;
};

struct Graph {
    int n_nos;
    struct Node* nos;
};

int main() {
    struct Graph *g = (struct Graph*)malloc(sizeof(struct Graph));
    g->n_nos = 3;

    g->nos = (struct Node*) malloc(sizeof(struct Node) * 3);
    g->nos[0].nome = "Santa Maria";
    g->nos[1].nome = "Cachoeira do Sul";
    g->nos[2].nome = "Frederico Westphalen";

    g->nos[0].n_vizinhos = 2;
    g->nos[1].n_vizinhos = 1;
    g->nos[2].n_vizinhos = 1;

g->nos[0].vizinhos = (struct Node*) malloc(sizeof(struct Node) * g->nos[0].n_vizinhos);
    g->nos[1].vizinhos = (struct Node*) malloc(sizeof(struct Node) * g->nos[1].n_vizinhos);
    g->nos[2].vizinhos = (struct Node*) malloc(sizeof(struct Node) * g->nos[2].n_vizinhos);

    g->nos[0].custos = (int*) malloc(sizeof(int) * g->nos[0].n_vizinhos);
    g->nos[1].custos = (int*) malloc(sizeof(int) * g->nos[1].n_vizinhos);
    g->nos[2].custos = (int*) malloc(sizeof(int) * g->nos[2].n_vizinhos);

g->nos[0].vizinhos[0] = g->nos[1];  // cachoeira do sul
    g->nos[0].vizinhos[1] = g->nos[2];  // frederico westphalen
    g->nos[1].vizinhos[0] = g->nos[0];  // santa maria
    g->nos[2].vizinhos[0] = g->nos[0];  // santa maria

    g->nos[0].custos[0] = 122;  // cachoeira do sul
    g->nos[0].custos[1] = 290;  // frederico westphalen
    g->nos[1].custos[0] = g->nos[0].custos[0];  // santa maria
    g->nos[2].custos[0] = g->nos[0].custos[1];  // santa maria

for(int i = 0; i < g->n_nos; i++) {
        printf(
            "O no %d se chama %s e possui %d vizinhos\n", i, 
            g->nos[i].nome, g->nos[i].n_vizinhos
        );
        for(int j = 0; j < g->nos[i].n_vizinhos; j++) {
            printf(
                "\tO vizinho %d do no %s se chama %s e fica a %dkm\n", j, 
                g->nos[i].nome, g->nos[i].vizinhos[j].nome, g->nos[i].custos[j]
            );
        }
    }

    return 0;
}

Implementao como matriz de adjacncias

A implementao em C como lista de adjacncias relativamente mais simples:

#include <stdio.h>
#include <stdlib.h>

int main() {
    char **nomes = (char**)malloc(sizeof(char*) * 3);
    nomes[0] = "Santa Maria";
    nomes[1] = "Cachoeira do Sul";
    nomes[2] = "Frederico Westphalen";

    int n_nos = 3;

    int matrix[3][3];

matrix[0][0] = 0;
    matrix[0][1] = 122;
    matrix[0][2] = 290;
    matrix[1][0] = 122;
    matrix[1][1] = 0;
    matrix[1][2] = -1;  // sem conexo
    matrix[2][0] = 290;
    matrix[2][1] = -1;  // sem conexo
    matrix[2][2] = 0;

for(int i = 0; i < n_nos; i++) {
        printf("O no %d se chama %s\n", i, nomes[i]);
        int n_vizinhos = 0;
        for(int j = 0; j < n_nos; j++) {
            if(matrix[i][j] > 0) {
                printf(
                    "\tO vizinho %d do no %s se chama %s e fica a %dkm\n",
                    n_vizinhos, nomes[i], nomes[j], matrix[i][j]
                );
                n_vizinhos += 1;
            }
        }
    }
    return 0;
}

rvores geradoras

Uma rvore geradora (spanning tree em ingls) uma estrutura no-tabular, derivada de um grafo no-direcionado, sem ciclos, onde todos os vrtices esto conectados uns aos outros por algumas de suas arestas.

rvores geradoras mnimas

Uma rvore geradora mnima (Minimum Spanning Tree ou MST em ingls) uma estrutura no-tabular, derivada de um grafo no-direcionado, sem ciclos, onde todos os vrtices esto conectados uns aos outros por algumas de suas arestas, e a soma do peso das arestas a menor soma possvel dentre todas as disponveis.

Aplicaes de rvores geradoras mnimas

Uma aplicao de rvores geradoras mnimas est na tarefa de agrupamento da rea de aprendizado de mquina.

Suponha que voc possua um conjunto de dados com duas variveis: a nota que um aluno tira em uma disciplina, e a porcentagem de presena do aluno na aula da disciplina. O conjunto de dados est distribudo no espao cartesiano da seguinte forma:

fig, ax = plot_graph(X)

O objetivo da tarefa de agrupamento colocar os pontos de dados (alunos) em grupos de maneira que cada ponto tenha mais caractersticas em comum com pontos do seu grupo do que pontos fora do seu grupo.

A qualidade dos arranjos (pontos em grupos) pode ser mensurada pela mtrica da silhueta, que vai de -1 1: quanto maior o valor desta mtrica, melhor.

Mas de que maneira arranjar os pontos em grupos?

Uma maneira de comear o problema gerar uma rvore geradora mnima com os pontos de dados. Para cada ponto, conecta-se ele com seu vizinho mais prximo.

Existem diversos algoritmos para gerar uma rvore mnima geradora. Para este exemplo, usaremos o Algoritmo de Prim:

  1. Escolha um vrtice arbitrrio para iniciar o processo
  2. Siga os passos 3 a 5 at que no existam vrtices a serem inclusos na MST
  3. Encontre arestas que conectem os vrtices da MST com os vrtices de fora da MST
  4. Dentre estes vrtices, escolha o de menor peso
  5. Adicione o vrtice MST, caso ele no gere um ciclo

def cheapest(dists, added, not_added):
    """Acha a aresta de menor peso"""
    cost = np.inf
    pivot = None
    cand = None
    for a in added:
        for n in not_added:
            if dists[a][n] <= cost:
                cost = dists[a][n]
                pivot = a
                cand = n

    return pivot, cand, cost

def find_mst(dists, added, not_added):
    G = nx.Graph()
    
    while len(not_added) > 0:
        pivot, cand, cost = cheapest(dists, added, not_added)
    
        G.add_edge(pivot, cand, weight=cost)
        
        print(f'{cand}', end=' ')
        added  = added.union({cand})
        not_added -= {cand}

    return G

E agora?

Desenhamos a rvore geradora mnima. Mas e agora?

Para o problema de agrupamento, precisamos remover arestas, calcular a mtrica da silhueta, e escolher o agrupamento gerado pela melhor configurao de remoes

No artigo PASCAL: An EDA for Parameterless Shape-Independent Clustering, foi utilizado um algortimo evolutivo para testar combinaes de remoes possveis

Nem todas as combinaes foram testadas, dado que este problema NP-difcil!

Para propsitos desta aula (e respeitando o escopo do assunto), nos limitaremos a remover apenas uma aresta por vez, diminuindo consideravelmente o espao de busca:

from sklearn.metrics import silhouette_score
import copy

scores = []
options = []
for a, b in G.edges():
    g = copy.deepcopy(G)
    g.remove_edge(a, b)
    group_a = nx.node_connected_component(g, a)
    group_b = nx.node_connected_component(g, b)
    labels = np.ones(10, dtype=int)
    labels[list(group_a)] = 0
    score = silhouette_score(X, labels)
    scores += [score]
    options += [(a, b)]

best = np.argmax(scores)
print(f'A aresta a ser removida gera uma silhueta de valor {scores[best]:.4f}')
